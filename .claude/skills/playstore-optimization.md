# Play Store Optimization

## Goal
Rewrite or improve a Play Store listing (title, short description, full description, keywords) to maximize organic discovery and conversion without sounding spammy.

## Instructions

### Step 1 — Gather Context
Ask if not provided:
- What does the app do? (core value in one sentence)
- Who is the target user? (busy professionals, students, families, etc.)
- What makes it different from competitors?
- Current title and descriptions (paste them in)
- Any known top competitor apps to benchmark against?

### Step 2 — Optimize the Title (max 30 characters)
Rules:
- Lead with the brand name, follow with a 1–2 word category keyword
- Format: `AppName – Keyword` or `AppName: Keyword`
- Avoid keyword stuffing — Google penalizes it
- Pick the single highest-volume keyword that fits naturally

### Step 3 — Short Description (max 80 characters)
Rules:
- Lead with the user benefit, not the feature list
- Use the most important secondary keyword naturally
- Create curiosity or convey immediate value
- End with a soft CTA if space allows ("Try free", "No account needed")

### Step 4 — Full Description (max 4000 characters)
Structure:
```
[Hook — 1-2 sentences: biggest user benefit]

[Feature highlights — 3-5 bullet points with ✓ or • prefix]

[Social proof or credibility — ratings, user count, press mentions]

[Secondary features — paragraph or list]

[Privacy/permissions note if relevant]

[CTA — Download now / Get started free]
```
Rules:
- Front-load keywords in the first 167 characters (above the fold)
- Repeat the top 3 keywords 3–5x naturally across the text
- Use plain language — 8th grade reading level
- Avoid ALL CAPS, excessive punctuation, or emoji overload

### Step 5 — Keyword Research (ASO)
Suggest keywords by category:
- **Core keywords**: what the app is (e.g., "expense tracker", "budget app")
- **Use-case keywords**: what problem it solves (e.g., "track spending", "monthly budget")
- **Audience keywords**: who uses it (e.g., "personal finance", "family budget")
- **Long-tail**: lower competition (e.g., "simple expense diary", "cash flow tracker")

Tip: Keywords in the title get 3× more weight than in the description.

### Step 6 — Conversion Tips
- Screenshots: first 2 frames must convey core value without reading
- Feature graphic: clear tagline, no clutter
- Icon: recognizable at 48×48px
- Ratings: respond to negative reviews publicly and professionally

## Context
- Indie developer positioning: emphasize simplicity, no subscription traps, privacy
- Avoid corporate/enterprise tone — users value authenticity
- Google indexes title + short description + full description for search

## Output Format
1. **Optimized title** (with character count)
2. **Short description** (with character count)
3. **Full description** (ready to paste)
4. **Keyword list** — grouped by category (Core / Use-case / Audience / Long-tail)
5. **Notes** — any assumptions made, A/B test suggestions, or areas needing more info
